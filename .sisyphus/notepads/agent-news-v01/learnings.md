## 2026-05-11 Task 5 CNInfo announcement adapter
- Created `myQuant/news_ingestion/sources/cninfo.py` with `CninfoAnnouncementSource(BaseNewsSource)`, `source = Source.CNINFO`.
- Endpoints: org lookup via `GET /new/information/topSearch/query?keyWord={symbol}`, search via `POST /new/hisAnnouncement/query` with JSON body `{"seDate", "pageNum", "pageSize": 30, "tabName": "fulltext", "stock", "orgId"}`.
- Pagination loops while `hasMore=true` up to `MAX_PAGES=100`, breaking on first non-success response or JSON decode error.
- PDF extraction via injectable `pdf_extractor` callable: default tries `pdfplumber` → `PyPDF2` → returns `"skipped_no_pdf_extractor"`. Extractor failures return `"failed"`.
- `_parse_announcement()` handles `annDate` in `"%Y-%m-%d %H:%M:%S"` and `"%Y-%m-%d"` formats. `adjunctUrl` becomes `source_item_id`; full PDF URL constructed as `http://static.cninfo.com.cn/{adjunctUrl}`.
- `content_hash` computed from `title|content` via SHA256.
- Each `RawNewsItem`: `source=Source.CNINFO`, `source_category=SourceCategory.ANNOUNCEMENT`, `body_status` reflects PDF extraction state.
- Test file `test_source_cninfo.py` has 6 tests: metadata parsing, PDF failure non-blocking, pagination, no adjunct URL, org lookup failure resilience, search HTTP failure recording.
- Fake transport pattern: `PoliteHttpClient(transport=fake_transport, request_interval=0.0, sleeper=lambda s: None)` with url-based dispatch.
- Verification passed: `PYTHONDONTWRITEBYTECODE=1 conda run -n vnpy43 python -m pytest myQuant/news_ingestion/tests/test_source_cninfo.py -q` (6 passed) and full suite (27 passed).
- LSP diagnostics clean for both files.

## 2026-05-11 Task 6 CLS telegraph adapter
- Created `myQuant/news_ingestion/sources/cls.py` with `ClsTelegraphSource(BaseNewsSource)`, `source = Source.CLS_TELEGRAPH`.
- CLS signing is deterministic: ordered string `app=CailianpressWeb&last_time={ts}&os=web&rn={rn}&sv=7.7.5`, SHA1 hex, then MD5 hex of that SHA1 string.
- Endpoint params are stringified `app/os/sv/rn/last_time/sign`; tests use fake `PoliteHttpClient` transport and injectable clock, so normal pytest performs no live HTTP.
- Response parsing expects `data.roll_data` and treats schema drift as structured coverage failure (`Status.FAILED`, `coverage_status="failed"`, parse error, empty items).
- Cursor pagination starts from current timestamp, then uses the oldest parsed `ctime` as next `last_time`; stops on empty page, unchanged cursor, max pages, or past the query start.
- `RawNewsItem` normalization: `id` -> `source_item_id`, `title`, `content`, `ctime` -> `published_at`, `shareurl` -> `url`, `SourceCategory.FLASH`, `body_status="inline"`, SHA256 over `title + content`.
- Verification passed: `PYTHONDONTWRITEBYTECODE=1 conda run -n vnpy43 python -m pytest myQuant/news_ingestion/tests/test_source_cls.py -q` (3 passed) and full news ingestion suite (30 passed); LSP diagnostics clean for `cls.py` and `test_source_cls.py`.

## 2026-05-11 Task 7 Eastmoney news adapter
- Created `myQuant/news_ingestion/sources/eastmoney.py` with `EastmoneyNewsSource(BaseNewsSource)`, `source = Source.EASTMONEY`.
- Stock mode uses injectable `PoliteHttpClient.get()` against a public Eastmoney search endpoint with `keyword={symbol}` and normalizes stock-specific items as `SourceCategory.FINANCIAL_NEWS`.
- Keyword fallback uses `https://so.eastmoney.com/news/s` with `keyword={query.keywords[0]}` when stock mode yields no items; keyword items use `SourceCategory.UNKNOWN`.
- Parser is stdlib-only (`html.parser`, `json`, `re`) and accepts best-effort HTML, JSON, and JSONP-shaped responses. Missing `published_at` is non-fatal: item is retained with `body_status="missing_published_at"`, metadata warning `missing_published_at`, and partial coverage.
- Eastmoney coverage is always marked `coverage_status="partial"` / `date_range_coverage="partial"` because reliable date-range pagination was not claimed for v0.1.
- `content_hash` follows Task 7 acceptance: SHA256 over `title + content`.
- Verification passed: `PYTHONDONTWRITEBYTECODE=1 conda run -n vnpy43 python -m pytest myQuant/news_ingestion/tests/test_source_eastmoney.py -q` (2 passed) and full news ingestion suite (32 passed); LSP diagnostics clean for `eastmoney.py` and `test_source_eastmoney.py`.

## 2026-05-11 Task 8 recall/filter/dedupe
- Created storage-agnostic `myQuant.news_ingestion.recall` package with `RecallEngine` and recall-local `MappedNews` carrying `available_at` for lookahead-safe downstream LLM candidates.
- Recall semantics implemented by strength: low direct company/name/symbol/alias; medium adds product/upstream/downstream/risk and industry; high adds profile macro factors plus generic macro/policy hints only when profile macro factors match, avoiding blanket macro mapping to all stocks.
- Dedup order follows v0.1 policy: first `(source, source_item_id)`, then `(source, content_hash)`; near-duplicate keys are computed but retained for now.
- Availability policy uses `published_at + 5 minutes` for datetimes, `15:00:00` for date-only items, and skips mappings for unknown publish time; `fetched_at` is not used as availability.
