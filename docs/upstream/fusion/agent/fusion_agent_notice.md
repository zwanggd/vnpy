# 智策使用须知

`智策` 能够显著提升策略研究和代码整理效率，但它始终属于辅助工具能力。无论是逻辑说明、策略代码、检查结果、回测结果还是参数优化结论，都不应被视为可以直接替代用户判断的最终答案。

## AI输出的局限性

在使用智策时，应当默认接受以下事实：

- AI 输出可能存在错误、遗漏、偏差或表达不完整；
- 同一个问题在不同时间、不同模型或不同配置下，结果可能并不完全一致；
- 即使文字描述看起来合理，也不代表对应代码一定可运行或适用于当前场景；
- 即使代码可以运行，也不代表逻辑正确、结果可靠或适合实盘。

因此，智策最适合用来“辅助研究与整理”，而不是代替用户进行最终判断。

## 为什么不能直接用于实盘

将智策输出直接用于实盘，通常会面临以下风险：

- 策略逻辑可能与用户原始意图存在偏差；
- 代码虽能运行，但边界条件处理不充分；
- 回测结果依赖样本区间、参数设置和历史数据质量；
- 参数优化结果可能过拟合；
- 市场环境变化可能导致历史有效逻辑在未来失效。

因此，“能生成”“能运行”“回测结果好”都不等于“可以直接实盘”。

## 使用前必须完成的验证工作

在考虑实盘应用前，建议至少完成以下验证工作：

1. 人工复核策略逻辑是否准确；
2. 人工检查关键代码结构与参数含义；
3. 完成必要的代码检查；
4. 使用历史数据执行回测并理解结果；
5. 执行参数优化时避免只追求历史最优；
6. 配合 [风控管理](../strategy/fusion_risk.md) 设置必要限制；
7. 在更低风险环境中先做进一步观察和验证。

对入门用户来说，真正重要的不是“生成得有多快”，而是“验证是否足够充分”。

## 用户应承担的责任

在使用智策过程中，用户应自行承担以下责任：

- 对输入内容的真实性、合法性和适用性负责；
- 对生成逻辑、代码和结果进行独立审查；
- 对是否进入仿真或实盘运行作出独立判断；
- 对据此产生的交易行为和后果负责。

如果你尚未建立清晰的验证流程，建议先回到 [智策工作流](fusion_agent_workflow.md)、[回测与优化](../strategy/fusion_backtester.md) 和 [风控管理](../strategy/fusion_risk.md)，先把研究闭环补完整。
