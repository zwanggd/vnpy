# 智策简介

`智策` 是 VeighNa Fusion 中面向策略研究与代码辅助的重点能力。它不是一个独立的交易接口，也不是“自动赚钱工具”，而是帮助用户把策略想法逐步整理为逻辑说明、策略代码、检查结果、回测结果和参数优化结论的一套辅助工作流。

## 智策是什么

在 Fusion 中，`智策` 可以理解为一个围绕策略研究展开的工作台。用户输入自己的策略想法后，系统会按步骤推进，帮助完成：

- 策略逻辑整理；
- 逻辑确认；
- 代码生成；
- 代码检查；
- 回测验证；
- 参数优化；
- 结果汇总与复核。

因此，智策更适合作为“研究助手”和“流程组织工具”来理解，而不是替代用户判断的自动决策系统。

## 适合哪些用户

智策尤其适合以下类型的用户：

- 已经有一定交易想法，但不确定如何清晰表达成策略逻辑的用户；
- 希望在图形界面中完成策略研究闭环，而不是完全手写代码和脚本的用户；
- 希望借助 AI 提升策略整理、代码草稿和结果分析效率的用户；
- 愿意对输出结果进行复核、回测和风控验证的用户。

如果你希望完全跳过理解过程，直接依赖 AI 输出进行实盘操作，那么这并不是智策的正确使用方式。

## 可以帮助完成哪些工作

智策能够帮助用户完成的典型工作包括：

- 把一句自然语言想法整理成更清晰的策略说明；
- 生成更容易评审的策略逻辑结构；
- 生成或修改 CTA 策略代码草稿；
- 对生成代码进行检查并给出反馈；
- 串联回测和参数优化步骤；
- 汇总研究过程中形成的主要结果。

这些能力可以显著降低“从想法到第一版可验证策略”的门槛，但并不会替代用户对策略有效性、合规性和风险的判断。

## 与传统策略开发方式的关系

智策并不是对传统策略开发方式的替代，而是对其进行辅助。更合适的理解方式是：

- 传统方式强调用户自行定义逻辑、编写代码、回测和优化；
- 智策在这些步骤之间提供结构化辅助；
- 最终是否采用某个逻辑、代码或参数，仍应由用户自己决定。

对于入门用户来说，建议把智策与 [CTA策略](../strategy/fusion_cta.md)、[回测与优化](../strategy/fusion_backtester.md) 结合起来使用，而不是把智策单独当作完整交易系统。

在投研过程中，【智策投研】可以负责执行量化策略研究任务，【智策助手】可以负责回答VeighNa、CTA策略开发和当前任务相关的问题。

## 从哪里开始

在 Fusion 主界面中，可通过 `【智策】` 菜单进入以下入口：

- `智策工作台`
- `AI服务配置`
- `模型浏览器`

首次使用时，建议先阅读 [智策配置](fusion_agent_config.md)，再进入 [智策工作流](fusion_agent_workflow.md) 了解完整流程。
